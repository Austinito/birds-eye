Bird's Eye exposes an HTTP JSON API under `/api/*`.

- Base URL in local development: `http://127.0.0.1:3100`
- Content type: JSON for most endpoints
- Streaming: **NDJSON** (`application/x-ndjson`) for live session streaming
- Auth: **none**

For route wiring, see [`apps/server/src/index.ts`](../apps/server/src/index.ts).

## Conventions

### Errors

On failure, endpoints generally respond with:

```json
{ "error": "Human readable message" }
```

Status codes are typically:

- `400` for validation or operational errors
- `404` for missing workspaces or sessions

### Identifiers

- `workspaceId`: UUID generated by Bird's Eye
- `viewerId`: base64url-encoded session file path used by the UI to refer to a session

### Session discovery

Sessions are discovered by scanning JSONL session files in:

- `~/.birdseye/sessions`
- `~/.pi/agent/sessions`

and filtering for sessions whose header `cwd` matches the workspace path.

See also:

- [Architecture: data model and storage](architecture.md#data-model-and-storage)
- [Development: session storage and discovery](development.md#session-storage-and-discovery)

## Health and config

### GET `/api/health`

Returns:

```json
{ "ok": true }
```

### GET `/api/config`

Returns basic server config paths:

```json
{
  "birdsEyeHome": "/Users/you/.birdseye",
  "legacyPiHome": "/Users/you/.pi"
}
```

## Settings

Settings are stored on the server at `~/.birdseye/settings.json`.

### GET `/api/settings`

Returns the current settings object, or `{}` if no settings file exists.

Example:

```json
{
  "defaultWorkspace": "last",
  "defaultModelKey": "anthropic/claude-sonnet-4-5",
  "defaultThinkingLevel": "medium"
}
```

### POST `/api/settings`

Persists settings. Unknown keys are preserved.

Body example:

```json
{
  "defaultWorkspace": "last",
  "defaultModelKey": "openai/gpt-4o",
  "defaultThinkingLevel": "off"
}
```

Returns the merged settings object.

Notes:

- `theme` is also accepted as `light|dark`, although the main theme toggle is stored in browser localStorage.
- `defaultWorkspace` may be a `workspaceId` or `"last"`.
- `defaultThinkingLevel` must be one of `off|minimal|low|medium|high|xhigh`.

See [`apps/server/src/birdseye-settings.ts`](../apps/server/src/birdseye-settings.ts).

## File browser

### GET `/api/browse?path=<path>`

Lists subfolders of the requested directory. If `path` is omitted, the server uses the user home directory.

Response:

```json
{
  "current": "/Users/you",
  "folders": [
    { "name": "workspace", "path": "/Users/you/workspace" }
  ]
}
```

Security note: this endpoint discloses local filesystem structure. See [security.md](security.md#api-surface-security-relevant-behaviors).

## Migration

Bird's Eye can copy selected parts of a legacy Pi install into `~/.birdseye/pi-agent`.

### GET `/api/migration/status`

Returns a `PiMigrationStatus` payload used by the UI.

### POST `/api/migration/run`

Body:

```json
{ "categories": { "settings": true, "auth": true, "resources": true, "context": true } }
```

Returns the updated `PiMigrationStatus`.

### POST `/api/migration/skip`

Returns the updated `PiMigrationStatus`.

See also:

- [Development: migration state](development.md#migration-state)
- [Security: migrated Pi configuration](security.md#migrated-pi-configuration-credentials)

## Workspaces

### GET `/api/workspaces`

Returns all registered workspaces.

### POST `/api/workspaces`

Body:

```json
{ "path": "/absolute/path/to/repo", "name": "Optional Name" }
```

Returns the created workspace.

### GET `/api/workspaces/:workspaceId`

Returns a workspace and touches `lastOpenedAt`.

When the workspace path exists, the response may also include Git metadata:

```json
{
  "git": {
    "isRepo": true,
    "branch": "main",
    "rootPath": "/absolute/repo/root"
  }
}
```

If the workspace is not inside a Git worktree, `git.isRepo` is `false`.

### DELETE `/api/workspaces/:workspaceId`

Deletes the workspace registration.

Returns:

```json
{ "ok": true }
```

## Sessions

### GET `/api/workspaces/:workspaceId/sessions`

Returns session summaries for the workspace.

### GET `/api/workspaces/:workspaceId/sessions/:viewerId`

Returns a `SessionDetail` payload.

If the session file has not been persisted yet for a newly created live session, the server may return a fallback detail assembled from in-memory runtime state.

## Live sessions

A live session is a Pi SDK `AgentSession` that the server has opened and is holding in memory.

For implementation details, see [architecture.md#live-session-runtime-pi-sdk](architecture.md#live-session-runtime-pi-sdk) and [`apps/server/src/pi-live-session.ts`](../apps/server/src/pi-live-session.ts).

### GET `/api/workspaces/:workspaceId/sessions/live`

Returns summaries of live sessions currently held in memory for this workspace:

```json
[
  { "viewerId": "...", "active": true, "isStreaming": false }
]
```

### POST `/api/workspaces/:workspaceId/sessions/live`

Creates a new session file and opens it live.

Body:

```json
{ "modelKey": "anthropic/claude-sonnet-4-5", "thinkingLevel": "medium" }
```

Returns:

```json
{ "detail": { /* SessionDetail */ }, "runtime": { /* LiveSessionState */ } }
```

### GET `/api/workspaces/:workspaceId/sessions/:viewerId/live`

Returns live runtime state if the session is open, otherwise returns an inactive runtime.

```json
{ "runtime": { "active": false } }
```

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/live`

Opens an existing session file live, or reuses an already-open runtime.

Body:

```json
{ "modelKey": "openai/gpt-4o", "thinkingLevel": "low" }
```

Returns:

```json
{ "runtime": { /* LiveSessionState */ } }
```

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/live/message`

Sends a message without streaming. The request is rejected if the agent is already streaming.

Body:

```json
{ "message": "Hello", "modelKey": "...", "thinkingLevel": "medium" }
```

Returns:

```json
{ "detail": { /* SessionDetail */ }, "runtime": { /* LiveSessionState */ } }
```

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/live/message/stream`

Sends a message and streams incremental updates as **NDJSON**.

- Response `Content-Type`: `application/x-ndjson`
- Each line is a JSON object matching `LiveSessionStreamEvent`

Example stream:

```ndjson
{"type":"status","message":"Working…"}
{"type":"thinking-start"}
{"type":"thinking-delta","delta":"Considering…"}
{"type":"thinking-end"}
{"type":"tool-start","toolCallId":"abc","toolName":"read"}
{"type":"tool-delta","toolCallId":"abc","delta":"..."}
{"type":"tool-end","toolCallId":"abc","toolName":"read","isError":false,"result":"..."}
{"type":"text-delta","delta":"Here is the answer"}
{"type":"done","detail":{},"runtime":{}}
```

Event types:

- `text-delta` — assistant text stream
- `thinking-start|thinking-delta|thinking-end` — reasoning stream
- `tool-start|tool-delta|tool-end` — tool execution stream
- `queue-update` — steering or follow-up queue changed
- `status` — status message
- `done` — final detail and runtime snapshot
- `error` — error message

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/live/queue`

Queues a message while the agent is already working.

Body:

```json
{ "message": "Do X next", "mode": "followUp" }
```

`mode` must be one of:

- `steer`
- `followUp`

Returns:

```json
{ "runtime": { /* LiveSessionState */ } }
```

## Session rename and archive

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/rename`

Body:

```json
{ "title": "New name" }
```

Returns the updated `SessionDetail`.

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/rename/auto`

Auto-generates a short UI-safe title.

Returns the updated `SessionDetail`.

### POST `/api/workspaces/:workspaceId/sessions/:viewerId/archive`

Archives the session.

Returns:

```json
{ "ok": true }
```

### POST `/api/workspaces/:workspaceId/sessions/archive`

Archives multiple sessions.

Body:

```json
{ "viewerIds": ["...", "..."] }
```

Returns:

```json
{ "ok": true, "archived": ["..."], "errors": [{ "viewerId": "...", "error": "..." }] }
```
