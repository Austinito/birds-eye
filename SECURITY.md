# Security Policy

## Purpose and scope

This file is the public vulnerability disclosure entry point for the Bird's Eye repository.

- Use this file to report security issues responsibly.
- Use [docs/security.md](docs/security.md) for security posture, trust boundaries, and deployment guidance.

## Reporting a vulnerability

Please **do not** report security issues through public GitHub issues, pull requests, discussions, or public chat.

### Preferred: GitHub Security Advisories

If this repository is hosted on GitHub, use the built-in private reporting flow:

1. Open the repository **Security** tab
2. Click **Report a vulnerability**

This creates a private disclosure channel with the maintainers.

### If private reporting is not available

Contact the repository owner or maintainer privately through their primary contact method and include **“Bird's Eye security report”** in the subject.

## What to include

To help with triage, include:

- a clear description of the issue and potential impact
- minimal reproduction steps and environment details
- affected component(s), such as `apps/server` or `apps/web`
- relevant logs, stack traces, or proof-of-concept code
- whether the issue is exploitable by an unauthenticated user

## Coordinated disclosure

- We may ask for additional details to reproduce and validate the report.
- Please avoid public disclosure until a fix is available or a disclosure timeline is agreed.
